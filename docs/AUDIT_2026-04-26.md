## Findings (severity-tagged)

### Critical
- postcss CVE (`<8.5.10`, GHSA-qx2v-qp2m-jg93)
- No security headers (CSP, HSTS, X-CTO, X-Frame-Options, Referrer-Policy, Permissions-Policy)
- No `/privacy`, `/terms`, `/cookies`, `/accessibility` pages
- Placeholder contact emails (`@station8.example`)
- Broken Bookings CTA (`#bookings-form` anchor target doesn't exist)
- Placeholder Instagram link
- No favicon, no `robots.txt`, no `sitemap.xml`, no OG image, no `manifest.json`

### High
- No JSON-LD structured data
- No `app/error.tsx` / `app/not-found.tsx`
- No CI workflow
- No SECURITY.md / security.txt
- No `metadataBase` / canonical
- ~23 MB raw image assets in `public/` (deferred — placeholders only, pro photos coming)
- `viewport.maximumScale: 5` blocks zoom (508 issue)
- `aria-hidden="false"` on a decorative Divider
- No skip-to-content link
- No error monitoring wired (Sentry env keys exist, no code)

### Medium / Nice-to-have
- Sanity CMS scaffolded but not integrated (P3)
- Booking form not built (P6)
- GSAP + motion both included (potential overlap)
- `pattern-circles.png` 1.3 MB instead of SVG
- Test coverage minimal (1 unit, 1 e2e)
- Empty placeholder folders in `features/` and `lib/`

## What was done well (intentionally noted)

- TypeScript strict + extra-strict flags
- Biome unified lint + format with a11y rules
- Server/client env separation via Zod, no leaked secrets
- Native `<dialog>` modal with proper aria
- Reduced-motion as a first-class user toggle
- All `<Image>` use `next/image` with descriptive alt text
- Clean architecture: `design-system/` / `features/` / `motion/` / `lib/`
- `next/font` with `display: swap` (zero CLS)
- README clear and phase-aware
